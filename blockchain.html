SHRABONI BANERJEE
To-Do App using Solidity, Hardhat, and React

This is a simple To-Do List application that integrates Solidity smart contracts, deployed using Hardhat, with a React frontend for user interaction.

Industry:
Blockchain Development, Decentralized Applications (DApps), and Web3 Software Development.

Tools:

  • Smart Contracts: Solidity
  • Development & Testing: Hardhat
  • Frontend: React, Node.js
  • Wallet: MetaMask
  • IDE: VSCode

    • Problem Statement:

    Traditional To-Do applications rely on centralized databases, making them vulnerable to:

    • Data Manipulation & Loss: Users have no real ownership of their tasks, and data can be lost due to server failures or security breaches.
    • Lack of Transparency: No verifiable record of task modifications or completions. Users must trust the service provider.
    User Research

    Users seek a secure, transparent, and decentralized task management solution to ensure data integrity, ownership, and verifiable task history without relying on centralized services.

    Solution
    • Decentralized Task Management: Smart contracts on Ethereum (or Hardhat local network) ensure tasks are immutable and tamper-proof. Users have full control over their tasks without relying on a central authority.
    • Gasless Transactions or Layer-2 Solutions: Implement meta-transactions (Gasless UX) to reduce gas fees. This improves usability, making frequent task updates affordable.
    • Transparent and Verifiable Task History: Each task update is stored on-chain, ensuring an auditable history of modifications. Users can verify who created, updated, or completed a task, preventing disputes.
    Output

    A fully functional decentralized To-Do List DApp where users can securely add, update, and track tasks on the blockchain with MetaMask integration and a React-based UI.

    Future Improvements:

    Enhance UI with better styling, add "Mark as Completed" and task filtering, enable persistent storage, and improve error handling for a smoother user experience.
